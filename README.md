# Roomflow 
스터디룸 예약 관리 시스템 (학습용 프로젝트)

## 프로젝트 개요
공간 예약 서비스의 핵심 도메인 로직을 설계하고, Spring MVC의 계층별 책임 분리를 실습하기 위한 프로젝트입니다. 
단순한 기능 구현을 넘어, **확장 가능한 아키텍처 설계**와 **사용자 경험(UX) 개선**에 초점을 맞추어 개발하고 있습니다.

---

## 기술 스택
- **Language**: Java 21
- **Framework**: Spring Boot 3.4.3
- **Template Engine**: Thymeleaf
- **Library**: Lombok, Spring Validation
- **Storage**: In-Memory (HashMap) 추후 DB연결 예정 

---

## 아키텍처 및 패키지 구조
계층형 구조에서 **도메인 기반 패키지 구조**로 리팩토링하여 응집도를 높이고 도메인 간 결합도를 낮추었습니다.

- `com.roomflow.domain`: 핵심 비즈니스 로직 (User, Room, Reservation)
  - `Entity`, `Repository`, `Service` 레이어 포함
- `com.roomflow.web`: 사용자 인터페이스 및 웹 관련 기술
  - `Controller`, `DTO (Form)`, `Session`, `Interceptor`

## 학습 포인트
1. **계층별 책임 분리**: 컨트롤러에서 레포지토리를 직접 참조하던 방식에서 서비스 레이어를 거치는 구조로 개선.
2. **DTO 활용**: 도메인 엔티티와 웹 요청 객체를 분리하여 Mass Assignment 보안 위협 방지 및 데이터 순수성 유지.
3. **동기 기반 학습**: 구현 중 마주친 문제를 해결해나가며 동기 기반 학습을 통해 성장
